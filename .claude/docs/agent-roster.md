# Agent Roster — Claude Web Studio

## Tier 1 — Directors (Opus)

Strategic decision-makers who guide the overall project.

| Agent | Role | When to Use |
|-------|------|-------------|
| `@project-architect` | System architecture, tech stack, scalability | "What architecture for a multi-tenant SaaS?" |
| `@product-owner` | Product vision, features, user stories | "What features should be in the MVP?" |
| `@tech-lead` | Code quality, dev practices, PR reviews | "Review my pull request" |

## Tier 2 — Department Leads (Sonnet)

Domain experts who lead their area of the project.

| Agent | Role | When to Use |
|-------|------|-------------|
| `@frontend-lead` | Frontend architecture, components, state | "How should I structure my React app?" |
| `@backend-lead` | API design, services, server performance | "Design the backend for user management" |
| `@ux-lead` | UX strategy, design system, accessibility | "Design the onboarding flow" |
| `@devops-lead` | CI/CD, deployment, infrastructure | "Set up GitHub Actions pipeline" |
| `@database-lead` | Data modeling, queries, migrations | "Design the database schema" |
| `@qa-lead` | Test strategy, quality gates, automation | "What should I test before release?" |
| `@security-lead` | App security, OWASP, auth architecture | "Audit our authentication flow" |
